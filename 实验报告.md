### 1.6 分析问题

**Q1：Adam 和 SGD 的收敛速度有何差异？从实验结果中你观察到了什么？**

**A1：** 
- **Adam 收敛更快**：Adam 在约 15-20 个 epoch 就达到稳定，而 SGD 需要 25-30 个 epoch
- **Adam 更稳定**：Adam 的 loss 曲线更平滑，SGD 波动较大
- **最终精度**：Adam 的验证准确率 (98.50%) 明显高于 SGD (97.80%)
- **原因**：Adam 结合了 Momentum 和自适应学习率的优势，能自动调整每个参数的学习步长

**Q2：学习率对训练稳定性有什么影响？**

**A2：**
- **SGD(lr=0.01)**：学习率较大，初期 loss 下降快但波动明显
- **Adam(lr=0.001)**：学习率适中，训练过程平稳，收敛稳定
- **学习率过大**：可能导致震荡甚至发散
- **学习率过小**：收敛速度慢，容易陷入局部最优
- **最佳实践**：Adam 默认 lr=0.001 通常是最优选择

**Q3：Batch Size 对模型泛化能力有什么影响？**

**A3：**
- **Batch Size=64**：验证准确率 98.50%，训练更稳定
- **Batch Size=128**：验证准确率 98.60%，略有提升
- **小 Batch**：梯度噪声大，有一定正则化效果，但训练不稳定
- **大 Batch**：梯度估计更准确，训练快，但可能降低泛化能力
- **实验结论**：在本实验中，Batch Size=128 配合 Early Stopping 效果略好

**Q4：Early Stopping 是否有效防止了过拟合？**

**A4：**
- **有效**：Early Stopping 在验证集 loss 不再下降时自动停止训练
- **实验 3**：在 18 个 epoch 停止，避免了后续过拟合
- **实验 4**：在 26 个 epoch 停止，保存了最佳模型权重
- **优势**：无需手动确定 epoch 数，自动捕捉最佳模型状态
- **效果**：验证准确率提升约 0.5-1%

**Q5：数据增强是否提升了模型的泛化能力？为什么？**

**A5：**
- **显著提升**：实验 4 (99.35%) 比实验 3 (98.60%) 提升 0.75%
- **原因 1**：数据增强扩充了训练集（3 倍），模型见到更多样化的样本
- **原因 2**：Rotation、Translation 模拟了手写数字的自然变化
- **原因 3**：Zoom 和 Noise 增强了模型对尺度变化和噪声的鲁棒性
- **原因 4**：配合 BatchNorm，进一步提升了泛化能力
- **结论**：数据增强是提升 CNN 性能最有效的方法之一

## 实验总结

### 关键技术点

1. **CNN 架构设计**：3 层 Conv + BatchNorm + MaxPool，有效提取空间特征
2. **优化器选择**：Adam 优于 SGD，收敛更快、精度更高
3. **数据增强**：Rotation、Translation、Zoom、Noise 提升泛化能力 0.75%
4. **Early Stopping**：自动捕捉最佳模型，防止过拟合
5. **Web 部署**：Gradio 快速构建交互式 Web 应用
6. **HuggingFace Spaces**：免费云平台部署深度学习模型

### 实验收获

- 深入理解了 CNN 的工作原理和超参数调优方法
- 掌握了数据增强和 Early Stopping 等防止过拟合的技术
- 学会了使用 Gradio 快速部署深度学习模型
- 体验了从模型训练到 Web 部署的完整流程
