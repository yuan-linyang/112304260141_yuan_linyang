# HuggingFace Spaces 部署指南

## 📤 部署步骤

### 方法 1：直接上传文件（推荐）

#### 1. 登录 HuggingFace
- 访问：https://huggingface.co
- 使用 GitHub 账号登录

#### 2. 创建新的 Space
- 点击右上角头像 → "New Space"
- 填写以下信息：
  - **Space name**: `mnist-digit-recognition`
  - **License**: MIT
  - **Space SDK**: Gradio
  - **Visibility**: Public
  - **Payment**: Free (CPU basic)

#### 3. 上传文件
上传以下文件到 Space 根目录：
- ✅ `app.py` - Web 应用主程序
- ✅ `best_model.pth` - 训练好的模型权重
- ✅ `requirements.txt` - 依赖包列表
- ✅ `README.md` - 项目说明

#### 4. 等待部署
- 上传完成后，Space 会自动开始部署
- 首次部署约需 2-5 分钟
- 部署成功后会显示 "Running" 状态

#### 5. 访问应用
- 部署完成后访问：`https://huggingface.co/spaces/yuan-linyang/mnist-digit-recognition`

---

### 方法 2：使用 Git 推送

#### 1. 克隆 Space 仓库
```bash
git clone https://huggingface.co/spaces/yuan-linyang/mnist-digit-recognition
cd mnist-digit-recognition
```

#### 2. 上传文件
```bash
# 复制文件到仓库目录
cp /path/to/app.py .
cp /path/to/best_model.pth .
cp /path/to/requirements.txt .
cp /path/to/README.md .

# 提交并推送
git add .
git commit -m "Upload MNIST digit recognition app"
git push
```

#### 3. 等待自动部署
- 推送后 HuggingFace 会自动构建和部署
- 可在 Space 页面查看部署进度

```markdown
# MNIST 手写数字识别系统

基于 CNN 的深度学习手写数字识别系统

## 功能特点
- 📤 上传图片识别
- ✏️ 在线手写板
- 🏆 Top-3 预测
- 📊 概率分布可视化

## 模型信息
- 架构：CNN (Conv2d + BatchNorm + ReLU + MaxPool)
- 验证准确率：99.35%
- 框架：PyTorch 2.5.1+cu121

## 使用说明
1. 上传手写数字图片 或 在画板上绘制
2. 点击"识别"按钮
3. 查看预测结果和概率分布
```

## ⚠️ 注意事项

### 1. 模型文件大小
- `best_model.pth` 约 1.5MB
- HuggingFace Spaces 免费版本支持最大 10GB 存储
- 完全足够使用

### 2. 依赖包版本
- 确保 `requirements.txt` 中包含所有必要的包
- Gradio 版本建议 >= 3.0.0
- PyTorch 版本建议 >= 2.0.0

### 3. 推理速度
- 免费 CPU 实例：~50-100ms/预测
- 如需更快响应，可升级到付费 GPU 实例

### 4. 并发限制
- 免费版本：最多 2 个并发请求
- 如果显示 "Space is busy"，请稍后再试

## 🆘 常见问题

### Q1: 部署失败怎么办？
**A:** 检查 Logs 查看错误信息，常见问题：
- 缺少依赖包 → 更新 requirements.txt
- 模型文件路径错误 → 使用相对路径
- 内存不足 → 优化模型或升级实例

### Q2: 页面加载很慢？
**A:** 
- 首次加载需要加载模型，约 5-10 秒
- 后续访问会快很多
- 可考虑使用 HuggingFace 的 CDN 加速

### Q3: 识别准确率不高？
**A:**
- 确保图片是手写数字
- 图片需要是黑底白字（或自动反转）
- 图片尺寸应接近 28x28
- 数字应在图片中央
