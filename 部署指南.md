## 步骤 3: 配置 Git LFS（重要！）

由于模型文件 `best_model.pth` 较大（约 1.5MB），需要使用 Git LFS：

```bash
# 1. 安装 Git LFS
git lfs install

# 2. 跟踪 .pth 文件
git lfs track "*.pth"

# 3. 创建/更新 .gitattributes 文件
# 这会自动创建 .gitattributes 文件

# 4. 添加并重新提交
git add .gitattributes
git add .
git commit -m "Add Git LFS tracking for model weights"
git push
```

**验证 LFS 是否生效**：
```bash
# 查看 LFS 跟踪的文件
git lfs ls-files

# 应该看到类似输出：
# a1b2c3d4e5 - best_model.pth
```

## 常见问题解决

### ❌ 问题 1: 部署失败 - 模型文件未找到

**错误信息**: `FileNotFoundError: best_model.pth`

**解决方案**:
```bash
# 确保模型文件通过 LFS 上传
git lfs ls-files

# 如果没有显示 best_model.pth，重新跟踪
git lfs track "*.pth"
git add .gitattributes
git add best_model.pth
git commit -m "Fix: Add model file with LFS"
git push
```

### ❌ 问题 2: 端口冲突

**错误信息**: `Address already in use`

**解决方案**:
Render 会自动设置 PORT 环境变量，修改 `app_final.py`:

```python
if __name__ == "__main__":
    import os
    port = int(os.environ.get('PORT', 7860))
    demo.launch(server_name="0.0.0.0", server_port=port)
```

### ❌ 问题 3: 内存不足

**错误信息**: `OutOfMemoryError`

**解决方案**:
1. 优化模型大小
2. 使用 CPU 推理（移除 `.to(device)`）
3. 升级到 Render 付费套餐

### ❌ 问题 4: 访问速度慢

**原因**: Render 免费套餐有冷启动

**解决方案**:
1. 首次访问后保持页面打开
2. 使用 [Uptime Robot](https://uptimerobot.com/) 定期 ping 服务
3. 升级到付费套餐

### ❌ 问题 5: GitHub 推送失败

**错误信息**: `Permission denied`

**解决方案**:
```bash
# 重新设置远程仓库
git remote remove origin
git remote add origin https://github.com/yuan-linyang/112304260141_yuan_linyang.git
git push -u origin main
```
